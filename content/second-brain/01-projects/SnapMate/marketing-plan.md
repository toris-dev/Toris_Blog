## 1. 포지셔닝

### 한 줄 메시지

**「찍는 순간, 함께 보는 우리만의 앨범」**

### 가치 제안 (vs 기존 앱)

| 대안 | 한계 | Snapmate |
|------|------|----------|
| 카톡·DM | 전송 누락·정리 어려움 | 촬영 즉시 그룹 반영 |
| iCloud 공유 앨범 | 실시간감·관계 UX 약함 | 피드·리액션·추억 타임라인 |
| Instagram | 공개·SNS 피로 | 폐쇄형 그룹·모멘트 |

### 1차 타깃 (런칭)

**20~30대 커플** — 데이트·기념일 촬영 빈도 높음, 「사진 보내줘」 Pain 직접 해소.

### 2·3차 확장 (로드맵)

| 단계 | 타깃 | 메시지 |
|------|------|--------|
| 2단계 | 가족 | 육아·부모님·여행 가족 앨범 |
| 3단계 | 친구·여행 모임 | 단체 여행 사진 자동 통합 |

## 2. 마케팅 목표 (MVP 런칭 후 90일)

기획 KPI([service_plan.md](./service_plan.md) §13)를 런칭 초기에 현실적으로 조정한 목표입니다.

| 지표 | 정의 | 90일 목표 (초안) |
|------|------|------------------|
| **그룹 생성률** | 가입자 중 1개 이상 그룹 생성 | 50%+ |
| **초대 전환율** | 초대 발송 → 48h 내 참여 | 35%+ |
| **D7 리텐션** | 가입 후 7일 재방문 | 25%+ (커플 30% 목표) |
| **공유 활성** | DAU당 일 업로드 | 2장+ (커플 그룹 5장 stretch) |
| **바이럴 계수** | 초대로 유입된 신규 / 기존 MAU | 0.3+ (초기) |

측정: Firebase Analytics(설정 시), Firestore 집계, 스토어 다운로드, 설문(온보딩 1문항).

## 3. 핵심 메시지·카피

### 슬로건 후보

1. **찍으면, 이미 함께 있어요**
2. **보내지 말고, 같이 쌓아요**
3. **우리 방에만 쌓이는 오늘**

### 채널별 톤

| 채널 | 톤 | 예시 |
|------|-----|------|
| 인스타·릴스 | 감성·짧은 Hook | 「데이트 끝나고 사진 47장 보내기? 이제 안 해요」 |
| 스토어 | 신뢰·기능 | 즉시 공유·프라이빗·추억 타임라인 |
| 초대 카피(앱 내) | 친근 | 「Snapmate "OO"에 초대해요!」 (이미 구현) |

### 금지·주의

- 「인스타 대체」「전 세계 공개」 — 폐쇄형 포지션과 충돌
- 과장 AI·지도 기능 — MVP 미포함

## 4. 채널 전략

### 4.1 제품 내 바이럴 (최우선)

앱에 이미 있는 성장 루프를 마케팅과 맞춥니다.

```
커플 A 가입 → 커플룸 생성 → QR/링크 초대 → 커플 B 가입
     → 촬영·푸시 → 상대 재방문 → 습관화
```

| 레버 | 액션 |
|------|------|
| 초대 UX | 그룹룸·초대 화면에 「연인에게 보내기」 CTA 강조 (카피 A/B) |
| 온보딩 | 가입 직후 「첫 그룹 만들기」 3단계 (이름 → 타입 커플 → 초대) |
| 푸시 | 「OO님이 사진을 올렸어요」 — 재방문 (이미 Functions 연동) |
| 모멘트 | 그룹 내 공개 사진 → 친밀도·재방문 (프로필 진입점 유지) |

### 4.2 유료·오가닉 (런칭 0~8주)

| 채널 | 역할 | 예산 가이드 (소규모) |
|------|------|----------------------|
| **Instagram Reels / TikTok** | 데모 영상(촬영→상대 폰 알림) | 제작 0원(자체) + 부스트 30~50만원/월 테스트 |
| **네이버 블로그·카페** | 커플·웨딩·여행 커뮤니티 스토리 | 시간 투자, 광고 최소 |
| **ASO** | 스토어 키워드·스크린샷 | [deployment-checklist.md](./deployment-checklist.md) §4.3 |
| **인플루언서 (마이크로)** | 커플 일상 5~10만 팔로워 3~5명 | 건당 20~50만원 또는 제품 교환 |
| **PR** | 「커플 사진 전송 fatigue」 앵글 | 보도자료 1건 + 인터뷰 2건 목표 |

### 4.3 출시 후 확장 (8주~)

- 가족 키워드 SEO·콘텐츠 (육아·명절)
- 여행 시즌 캠페인 (여행룸 타입)
- 제휴: 스튜디오·웨딩·펜션 「Snapmate 방 만들고 촬영」

## 6. 콘텐츠·크리에이티브 계획

### 6.1 필수 제작물

| 자료 | 용도 | 담당 |
|------|------|------|
| 15~30초 데모 영상 | Reels, 스토어 프리뷰(선택) | 영상 |
| 스토어 스크린샷 5~8장 | ASO | 디자인 ([deployment-checklist.md](./deployment-checklist.md)) |
| OG 이미지 1200×630 | 링크 공유·랜딩 | 디자인 |
| 초대 랜딩 1페이지 | `join?code=` 설명 + 스토어 배지 | 웹 |

### 6.2 콘텐츠 테마 (4주 로테이션 예시)

| 주 | 테마 | Hook |
|----|------|------|
| 1 | 데이트 | 「카톡 사진 정리 그만」 |
| 2 | 기념일 | 「1년 전 오늘, 앱이 먼저 보여줘요」 |
| 3 | 여행 | 「각자 찍어도 한 앨범」 |
| 4 | 프라이버시 | 「우리 둘만 보는 방」 |

### 6.3 UGC·커뮤니티

- 해시태그: `#Snapmate` `#찍으면함께` (통일)
- 커플 챌린지: 「일주일 매일 1장」 — 상품은 후속(스킨·용량)
- 피드백: Google Form / Discord 소규모 베타

## 9. 성과 측정·실험

### 9.1 주간 대시보드 (최소)

| 항목 | 소스 |
|------|------|
| 신규 가입 | Firebase Auth / Analytics |
| 그룹 생성·초대 발송·참여 | Firestore 집계 |

| 일 업로드 수 | `photos` 생성 수 |
| 스토어 전환 | Connect / Play 설치·페이지 조회 |

1. 온보딩: 「커플룸 먼저」 vs 「초대 받고 참여」
2. 홈 CTA: 「찍고 바로 공유」 vs 「우리 방 들어가기」
3. 스토어 스크린샷 1번 이미지 (카메라 vs 피드)
