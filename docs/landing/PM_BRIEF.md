# Toris Blog /projects/[slug] 랜딩페이지 재구축 — 디자인 요구사항 브리프
**작성**: PM | **대상**: 디자이너 2, FE 2

## 0. 미션
포트폴리오 → **각 서비스의 실제 프로덕션 마케팅 랜딩**. 방문자 = 잠재 사용자. 목표 = 전환.

## 1. 공통 요구사항
- 벤치마크: Apple 제품 페이지, busy.app, letsjive.io
- framer-motion 필수: 섹션 scroll-reveal + useScroll/useTransform 스크롤 연동 최소 1개
- 스크롤 내러티브: 히어로 → 문제 → 해결/기능 → 작동 방식 → 신뢰 → 최종 CTA
- 마이크로 인터랙션: hover/press 피드백, 카드 lift
- 프로덕트 목업 CSS/SVG 필수: 모바일=폰 프레임, 데스크톱=OS 윈도우, 웹=브라우저, CLI=터미널
- 라이트/다크 페어 필수 (bg-slate-50 dark:bg-[#050810] 패턴)
- prefers-reduced-motion 존중, 이모지 아이콘 금지(SVG만)
- accent 컬러 일관 적용, 8페이지 템플릿 복붙 금지
- 히어로 1차 CTA = 서비스 전환 액션 (개발 중 = 사전 등록/출시 알림)

## 2. 서비스별 (요약)
- love-trip: 로맨틱 로즈, 브라우저 목업(플래너), 폼→지도 전환, 1/N 카운트업
- tracedesk: 인디고 신뢰, OS 윈도우 목업, 타임라인 채워짐, 점수 게이지
- snapmate: 앰버 즉흥, 폰 목업, 셔터→갤러리 착지, 2대 동기화
- devpulse: 그린 해커, 터미널 타이핑, 파이프라인 노드 점등, 비용 비교
- loca: 시안 도시, 폰 목업 채팅 버블 순차 등장, 500m 반경 펄스
- bubble-bible: 골드 성장, 폰 목업, 레벨 프로그레스/뱃지 해금, 포인트 카운터
- pepebear: 네온 그린 에너지, 라이브 카운터/순위 셔플, 페이즈 타임라인
- yeti: 아이스 블루 레트로, 펭귄 포물선 SVG, 로드맵 스테이지 해금

## 3. 반려 기준
1. 정적(스크롤 애니메이션/호버 피드백 없음)
2. 포트폴리오처럼 보임(기술스택/GitHub이 주인공 — devpulse CTA만 예외)
3. 목업 부재
4. 히어로 전환 CTA 부재
5. 템플릿 복붙(고유 인터랙션 1개 + 고유 섹션 연출 1개 미달)
6. 라이트/다크 페어 누락, reduced-motion 미대응, 이모지 아이콘
