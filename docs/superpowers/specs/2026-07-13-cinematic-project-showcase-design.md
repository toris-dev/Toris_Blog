# 신규 프로젝트 시네마틱 쇼케이스 설계

## 목적

`/projects`가 GitHub 저장소 목록을 보여주는 데서 끝나지 않고, 방문자가 각 제품의 핵심 행동을 직접 체험한 뒤 프로젝트의 가치와 제작 역량을 이해하도록 만든다.

대상 독자는 협업 파트너, 채용 담당자, 초기 사용자와 오픈소스 기여자다. 각 랜딩의 단일 목적은 “이 프로젝트가 무엇을 해결하고 어떤 경험을 제공하는지 30초 안에 이해시키는 것”이다.

## 범위

기존 16개 전용 랜딩은 유지하고 다음 9개를 추가한다.

1. 21n Apps
2. SnapMate
3. Bubble Bible
4. 동네 칠하기 대작전
5. 청년머니가이드
6. 별빛 온실
7. 30초 배구왕
8. toris-docs
9. product-growth-skills

`/Users/toris/projects/build`, `/Users/toris/projects/supabase`, 편집기 설정 폴더는 독립 제품이 아니므로 제외한다. `21n Apps`는 공개 README의 제품·아키텍처 정보만 사용하며 병원, 모델, 사용자 mock 사진은 사용하지 않는다.

## 정보 출처

- 로컬 저장소의 README, 제품·게임 디자인 문서, 브랜드 토큰, 스토어 에셋과 실제 화면
- GitHub 커넥터에서 확인한 공개 저장소 메타데이터와 README
- 기존 `src/data/projects.ts`, `src/components/projects/landing/index.ts`, 공통 랜딩 프리미티브

불확실한 수치, 출시 상태, 성과 지표는 만들지 않는다. README 또는 코드에서 확인되지 않은 기능은 카피에 넣지 않는다.

## 구조

### 데이터

`src/data/projects.ts`에 9개 `Project` 항목을 추가한다. 각 항목은 고유 slug, 실제 설명, 플랫폼, 상태, 태그, 기술, 기능과 이미지 경로를 가진다.

`product-growth-skills`는 공개 GitHub 저장소이므로 Repository Atlas에도 추가한다. 나머지 비공개 저장소는 전용 프로젝트 카드와 랜딩에서 노출하되 GitHub 링크가 방문자에게 404를 만들 경우 프로필 링크 또는 공개 서비스 링크를 사용한다.

### 라우팅

기존 `/projects/[slug]` 정적 생성 방식을 유지한다. `LANDINGS` 레지스트리에 9개 컴포넌트를 동적 import로 등록한다. 일반 상세 페이지로 폴백하지 않으며, 모든 신규 프로젝트는 전용 랜딩을 가진다.

### 파일 경계

- 프로젝트별 랜딩: `src/components/projects/landing/<Project>Landing.tsx`
- 공통 시네마틱 프리미티브: 기존 `shared.tsx`를 확장하거나 별도 `cinematic-shared.tsx`로 분리
- 실제 마케팅 자산: `public/images/projects/<slug>/`
- 프로젝트 데이터: `src/data/projects.ts`
- 전체 저장소 색인: `src/data/githubRepositories.ts`

프로젝트별 파일은 자기 인터랙션과 화면 서사만 소유한다. 공통 Reveal, CTA, reduced-motion 처리, 기기 프레임과 접근성 헬퍼만 공유한다.

## 시각·인터랙션 방향

각 페이지는 하나의 시그니처 인터랙션에만 시각적 대담함을 집중한다. 나머지 섹션은 읽기 쉬운 정보 구조와 실제 제품 에셋으로 정돈한다.

### 21n Apps

- 대상: 병원 운영자, 모델과 의료 플랫폼 파트너
- 팔레트: Clinical Navy `#172554`, Contract Blue `#2563EB`, Signature Coral `#FB7185`, Sterile Mist `#F8FAFC`, Verified Mint `#34D399`
- 타이포그래피: 굵은 산세리프 제목, 중립적 본문, 계약 상태용 모노스페이스
- 시그니처: Draft → 모델 서명 → 병원 확인 → 체결 완료를 진행하는 전자계약 타임라인
- 금지: 환자·병원 mock 사진, 실제 개인정보처럼 보이는 데이터

### SnapMate

- 대상: 커플, 가족, 친구 그룹
- 팔레트: Warm Cream `#FFF7ED`, Film Brown `#4A2D24`, Sunset Peach `#FB923C`, Memory Rose `#FB7185`, Cloud White `#FFFFFF`
- 타이포그래피: 둥근 디스플레이 제목, 편안한 본문, 촬영 시간용 모노스페이스
- 시그니처: 셔터를 누르면 사진이 현상되어 실시간 그룹 갤러리로 이동
- 실제 스토어 feature graphic과 화면을 사용한다.

### Bubble Bible

- 대상: 성경 읽기 습관과 교회·소그룹 나눔을 원하는 사용자
- 팔레트: Parchment `#FFF8E7`, Cocoa `#4A3326`, Scripture Gold `#C99A36`, Olive `#6B7A3A`, Prayer Blue `#7393B3`
- 타이포그래피: 구절용 명조 계열, 본문용 산세리프, 장·절 표기용 모노스페이스
- 시그니처: 오늘의 구절을 넘기고 읽기 완료를 누르면 연속 기록과 그룹 나눔이 이어지는 흐름
- 교리·번역 우월성 표현 없이 제품 기능만 설명한다.

### 동네 칠하기 대작전

- 대상: 짧고 경쟁적인 모바일 캐주얼 게임 사용자
- 팔레트: Asphalt `#263238`, Alley Cream `#FFF3D6`, Territory Jade `#18B87A`, Delivery Coral `#FF6B4A`, Coin Yellow `#FFD447`
- 타이포그래피: 굵고 압축된 게임 제목, 짧은 UI 카피, 점수용 모노스페이스
- 시그니처: 드래그한 trail이 출발 영역으로 돌아오면 내부 타일이 색칠되는 미니 점령판
- 서버 멀티플레이처럼 표현하지 않고 AI 봇 대전과 로컬 저장을 명시한다.

### 청년머니가이드

## 공통 페이지 서사

1. 제품의 대표 행동을 직접 체험하는 히어로
2. 해결하려는 실제 문제와 사용 맥락
3. 핵심 흐름 또는 시스템 구조
4. 실제 화면·에셋 기반 기능 쇼케이스
5. 검증 가능한 기술·운영 결정
6. GitHub, 서비스, 스토어 또는 사전등록 CTA

모든 프로젝트에 같은 섹션 수를 강제하지 않는다. 확인 가능한 내용이 적은 프로젝트는 짧게 끝낸다.

## 모션과 접근성

- 페이지 로드에서 한 번, 대표 인터랙션 완료에서 한 번만 강한 모션을 사용한다.
- `prefers-reduced-motion`에서는 반복·패럴랙스·물리 애니메이션을 정적 상태로 대체한다.
- 모든 조작은 버튼 또는 명시적 키보드 포커스를 가진다.
- 색상만으로 상태를 전달하지 않고 텍스트·아이콘·형태를 함께 사용한다.
- 모바일에서 44px 이상의 터치 영역을 유지한다.
- 자동 재생 영상이나 소리를 추가하지 않는다.

## 자산 처리

- 로컬 저장소의 검증된 스토어 이미지, 로고, 화면 캡처를 우선 사용한다.
- 외부 GitHub OpenGraph 이미지는 요청 제한에 대비해 브랜드 폴백을 제공한다.
- 원본 프로젝트 파일을 수정하지 않고 필요한 자산만 Toris Blog의 `public/images/projects/`로 복사한다.
- 큰 PNG는 시각 품질을 유지하는 범위에서 WebP 또는 최적화 PNG로 변환한다.
- 사용자·병원·개인정보처럼 보이는 mock 이미지는 제외한다.

## 오류와 경계 상태

- 이미지 로드 실패: 프로젝트 팔레트와 이름으로 만든 폴백 그래픽 표시
- 공개 URL 없음: GitHub 프로필 또는 사전등록 CTA 사용, 존재하지 않는 라이브 링크를 만들지 않음
- 인터랙션 미완료: 다음 가능한 행동을 짧은 문장으로 안내
- reduced-motion: 즉시 완료 가능한 정적 인터랙션 제공
- 작은 화면: 인터랙션을 축소하거나 단계형 컨트롤로 변환하며 가로 스크롤에 의존하지 않음

## 검증

### 정적 검증

- 프로젝트 데이터 9개와 `LANDINGS` 9개 slug가 정확히 일치
- `/projects/[slug]`의 `generateStaticParams`에 9개 경로 포함
- 변경 파일 대상 TypeScript 검사
- `next build --webpack` 성공
- `git diff --check` 성공

### 브라우저 검증

- `/projects`에서 9개 카드가 노출되고 필터 수가 갱신됨
- 모바일 375px, 태블릿 768px, 데스크톱 1280px에서 주요 섹션 캡처
- 각 랜딩의 대표 인터랙션 시작·진행·완료 상태 확인
- 키보드 Tab 이동과 focus-visible 확인
- reduced-motion 에뮬레이션에서 반복 모션 중단 확인
- 콘솔에서 신규 런타임 오류가 없음

기존 MongoDB 환경 변수 부재나 GitHub 이미지 요청 제한처럼 신규 변경과 무관한 로컬 오류는 별도로 기록하되, 신규 랜딩 검증 결과와 섞지 않는다.

## 완료 조건

- 9개 신규 프로젝트가 `/projects` 카드와 전용 정적 랜딩을 가진다.
- 각 랜딩은 프로젝트 고유 시그니처 인터랙션을 제공한다.
- 확인된 프로젝트 정보와 실제 자산만 사용한다.
- 21n Apps와 toris-docs에서 민감하거나 비공개인 내용을 노출하지 않는다.
- 접근성, reduced-motion, 이미지 폴백과 반응형 레이아웃이 검증된다.
- 타입 검사, 프로덕션 빌드와 브라우저 QA가 통과한다.
