# lifestyle_blog 프로젝트 리뷰 - 한국어 라이프스타일 블로그 플랫폼

## 프로젝트 개요

**프로젝트명**: lifestyle_blog  
**기간**: 2025.12  
**기술 스택**: Next.js, TypeScript, Tailwind CSS, Markdown, Google Analytics  
**목적**: 현대적인 한국어 라이프스타일 블로그 플랫폼 구축

lifestyle_blog는 제가 기술 블로그와는 다른, 일상과 라이프스타일을 공유하는 블로그를 만들고 싶어서 시작한 프로젝트입니다. 마크다운 기반 콘텐츠 관리와 파트너스 프로그램 통합이 특징입니다.

## 프로젝트를 시작한 이유

기술 블로그(Toris Blog)는 개발 관련 내용만 다루는데, 일상이나 라이프스타일 같은 내용도 공유하고 싶었습니다. 그리고 블로그를 통해 수익을 얻을 수 있는 방법도 고민하게 되었습니다.

그래서 "한국어로 작성하기 편하고, 수익화도 가능한 라이프스타일 블로그"를 만들고 싶었습니다.

## 주요 기능 구현

### 1. 마크다운 기반 CMS

파일 기반 CMS를 구현하여 마크다운으로 콘텐츠를 관리했습니다.

**처음 접했을 때:**

- "CMS가 뭐지?"라는 생각부터 시작했습니다.
- 마크다운을 HTML로 변환하는 것이 처음에는 어려웠습니다.
- 프론트매터(frontmatter)를 파싱하는 것이 까다로웠습니다.

**구현:**

```typescript
// 마크다운 파싱
import matter from 'gray-matter';
import { remark } from 'remark';
import html from 'remark-html';

const parseMarkdown = (content: string) => {
  const { data, content: markdown } = matter(content);
  const processedContent = remark().use(html).processSync(markdown);
  return {
    frontmatter: data,
    content: processedContent.toString()
  };
};
```

**배운 점:**

- 마크다운은 작성하기 편하고, HTML로 변환하기도 쉽습니다.
- 프론트매터를 활용하면 메타데이터를 쉽게 관리할 수 있습니다.
- 파일 기반 CMS는 버전 관리가 쉽습니다.

### 2. 쿠팡 파트너스 통합

쿠팡 파트너스 상품 카드를 자동으로 삽입하는 기능을 구현했습니다.

**처음 접했을 때:**

- "파트너스 프로그램이 뭐지?"라는 생각부터 시작했습니다.
- 쿠팡 API를 사용하는 것이 처음에는 어려웠습니다.
- 상품 카드를 어떻게 표시할지 고민이 많았습니다.

**구현:**

```typescript
// 쿠팡 상품 카드 컴포넌트
interface CoupangProductProps {
  productId: string;
  title: string;
}

const CoupangProduct = ({ productId, title }: CoupangProductProps) => {
  return (
    <div className="coupang-product-card">
      <iframe
        src={`https://coupa.ng/${productId}`}
        width="100%"
        height="200"
        frameBorder="0"
        scrolling="no"
      />
    </div>
  );
};
```

**배운 점:**

- 파트너스 프로그램은 블로그 수익화의 좋은 방법입니다.
- 사용자 경험을 해치지 않도록 적절히 배치해야 합니다.
- 법적 고지사항을 명시해야 합니다.

### 3. 조회수 및 좋아요 통계

마크다운 파일에 통계를 자동으로 저장하는 시스템을 구현했습니다.

**구현 과정:**

- 사용자가 포스트를 조회하면 조회수 증가
- 좋아요 버튼 클릭 시 좋아요 수 증가
- 마크다운 파일의 프론트매터에 자동으로 저장

**어려웠던 점:**

- 파일을 직접 수정하는 것이 보안상 위험할 수 있었습니다.
- 동시성 문제를 고려해야 했습니다.
- 파일 시스템 접근 권한 문제가 있었습니다.

**해결 방법:**

- 서버 사이드에서만 파일을 수정하도록 했습니다.
- 파일 잠금을 활용하여 동시성 문제를 해결했습니다.
- 백업을 자동으로 생성했습니다.

### 4. 실시간 미리보기 에디터

마크다운을 작성하면서 실시간으로 미리보기를 볼 수 있는 에디터를 구현했습니다.

**구현:**

- 좌측: 마크다운 에디터
- 우측: 실시간 미리보기
- 동기화된 스크롤

**배운 점:**

- 실시간 미리보기는 작성 경험을 크게 향상시킵니다.
- 성능을 고려해야 합니다. 너무 자주 업데이트하면 느려질 수 있습니다.
- 디바운싱을 활용하여 성능을 최적화했습니다.

### 5. SEO 최적화

SEO 최적화를 위해 다양한 기능을 구현했습니다.

**구현:**

- Open Graph 태그
- JSON-LD 구조화 데이터
- 사이트맵 자동 생성
- robots.txt

**배운 점:**

- SEO는 검색 엔진에서 노출되는 데 중요합니다.
- 구조화된 데이터를 추가하면 검색 결과가 더 예쁘게 표시됩니다.
- 사이트맵을 생성하면 검색 엔진이 더 쉽게 크롤링할 수 있습니다.

## 어려웠던 점

### 1. 파일 시스템 접근

마크다운 파일을 직접 수정하는 것이 보안상 위험할 수 있었습니다.

**문제점:**

- 서버에서 파일을 직접 수정하면 보안 문제가 발생할 수 있습니다.
- 파일 시스템 접근 권한을 관리해야 합니다.
- 에러가 발생하면 파일이 손상될 수 있습니다.

**해결 방법:**

- 파일 수정은 서버 사이드에서만 수행하도록 했습니다.
- 권한을 엄격하게 관리했습니다.
- 백업을 자동으로 생성하여 데이터 손실을 방지했습니다.

### 2. 한글 타이포그래피

한글 콘텐츠를 예쁘게 표시하는 것이 중요했습니다.

**고려사항:**

- 한글 폰트 선택
- 줄간격 및 자간 조정
- 가독성 향상

**해결 방법:**

- 한글에 최적화된 폰트를 선택했습니다.
- 적절한 줄간격과 자간을 설정했습니다.
- 모바일에서도 읽기 편하도록 반응형 타이포그래피를 적용했습니다.

### 3. 성능 최적화

마크다운 파일이 많아지면 성능 이슈가 발생할 수 있었습니다.

**문제점:**

- 모든 마크다운 파일을 파싱하면 느려질 수 있습니다.
- 이미지가 많으면 로딩이 느려집니다.

**해결 방법:**

- 필요한 파일만 파싱하도록 최적화했습니다.
- 이미지를 최적화하고 lazy loading을 적용했습니다.
- 캐싱을 활용하여 성능을 개선했습니다.

## 개선하고 싶은 점

### 1. 관리자 페이지

현재는 파일을 직접 수정해야 합니다. 관리자 페이지를 만들어서 웹에서 직접 수정할 수 있도록 하고 싶습니다.

### 2. 댓글 시스템

현재는 댓글 기능이 없습니다. 댓글 시스템을 추가하여 독자와 소통하고 싶습니다.

### 3. 검색 기능 개선

현재 검색 기능이 기본적입니다. 더 정확하고 빠른 검색을 구현하고 싶습니다.

## 총평

lifestyle_blog는 제가 기술 블로그와는 다른 방향으로 만든 블로그입니다. 마크다운 기반 CMS를 구현하고, 파트너스 프로그램을 통합하며, 통계 시스템을 만드는 경험을 할 수 있어서 뿌듯합니다.

특히 파일 기반 CMS를 구현하면서 "콘텐츠를 어떻게 효율적으로 관리할까?"를 고민하게 되었고, 파트너스 프로그램을 통합하면서 블로그 수익화에 대한 이해도 높일 수 있었습니다.

한글 콘텐츠를 예쁘게 표시하는 것도 중요하다는 것을 배웠고, SEO 최적화를 통해 검색 엔진에서 노출되는 방법도 배울 수 있었습니다.
